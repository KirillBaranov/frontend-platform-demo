# ADR-002: Contracts Layer

## Статус
Принято

## Контекст
В micro-frontend архитектуре модули должны общаться друг с другом, но не зависеть друг от друга. Прямые импорты между модулями создают связанность: изменение в модуле A ломает модуль B.

При этом модули могут быть на разных фреймворках и обновляться независимо. Нужен стабильный слой коммуникации.

## Решение
Выделен пакет `@platform/contracts` — единственный пакет, который импортируют все модули. Содержит:

- **Events** (версионированные) — "что произошло" (OrderCreated, VehicleSelected)
- **Commands** (версионированные) — "сделай это" (NavigateTo, ShowNotification)
- **ModuleDefinition** — интерфейс mount/unmount для каждого модуля
- **Zod schemas** — валидация данных на границах (Vehicle, Order, DashboardStats)
- **Feature flags** — типы и дефолтные значения

### Версионирование

Каждый event/command имеет поле `version`. Когда контракт эволюционирует:

```typescript
// v1 — исходный
interface VehicleSelectedV1 { version: 1; vehicleId: string; brand: string; model: string; }

// v2 — добавлены поля price, year
interface VehicleSelectedV2 { version: 2; vehicleId: string; brand: string; model: string; price: number; year: number; }
```

Старые модули продолжают работать с v1, новые используют v2. Shell может роутить оба одновременно. Модули мигрируют независимо.

## Последствия

### Положительные
- Модули изолированы — знают только контракты, не друг друга
- Замена модуля = реализация того же контракта в новом коде
- Типобезопасность на уровне компиляции (TypeScript)
- Валидация данных на границах (Zod) предотвращает runtime ошибки
- Версионирование позволяет эволюцию без breaking changes

### Отрицательные
- Дополнительный пакет для поддержки
- Изменение контракта требует координации (но меньше, чем без контрактов)

## Альтернативы

| Вариант | Почему отклонён |
|---------|-----------------|
| Custom Events (DOM) | Нет типизации, нет версионирования |
| postMessage | Для iframe, избыточно для SPA модулей |
| Redux/Pinia (shared store) | Lock-in на конкретный state manager |
