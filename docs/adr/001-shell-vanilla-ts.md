# ADR-001: Shell на vanilla TypeScript

## Статус
Принято

## Контекст
Система должна работать десятилетия. За последние 10 лет индустрия прошла через jQuery → Angular 1 → React → Vue → Svelte → Solid. Средний срок жизни популярного фреймворка — 3-7 лет.

Shell — это оркестратор, который загружает модули, управляет роутингом, event bus, feature flags. Если shell привязан к фреймворку, его замена = переписывание всей инфраструктуры.

## Решение
Shell написан на vanilla TypeScript. Использует только стабильные Web API: DOM, fetch, History API, CustomEvent.

Ноль зависимостей от React, Vue, Angular или любого другого фреймворка.

## Последствия

### Положительные
- Shell не требует миграции при смене фреймворка модулей
- Нет vendor lock-in на уровне оркестратора
- Минимальный размер бандла (~48KB)
- Понятен любому разработчику, независимо от стека

### Отрицательные
- Больше boilerplate чем с фреймворком (ручная работа с DOM)
- Нет готовой экосистемы компонентов для shell UI (sidebar, header)
- Требует дисциплины: не тянуть фреймворк "для удобства"

## Альтернативы

| Вариант | Почему отклонён |
|---------|-----------------|
| Shell на React | Lock-in. Через 5 лет возможно потребуется миграция shell |
| Shell на Lit (Web Components) | Лучше чем React, но добавляет зависимость от Lit runtime |
| Shell на Stencil | Генерирует Web Components, но сам является фреймворком |
